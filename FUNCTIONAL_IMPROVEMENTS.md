# 官网功能性优化建议（参考高端 SaaS / AI 产品站）

基于 2024–2026 年 SaaS / AI 产品官网的常见做法与转化研究整理，对应到当前页面，便于排期或开 Issue。

---

## 一、行业常见做法摘要

| 做法 | 说明与数据参考 |
|------|----------------|
| **首屏 5 秒定生死** | 首屏需在 5 秒内传达「是什么、为谁、结果如何」；加载慢或信息模糊会直接拉高跳出率。 |
| **互动演示 > 静态截图** | 首屏或首屏下方用**可互动/自动播放的产品演示**，比静态截图更能提升试用转化（约 3.2x）。 |
| **首屏信任密度** | 每屏至少 3 个信任信号：客户数、Logo 墙、评价、安全标识、合作伙伴等，减少首屏内的疑虑。 |
| **主 CTA 文案** | 「免费开始 / Start free」类 CTA 在 PLG 场景下通常优于「预约演示」；主按钮需明确、可行动。 |
| **定价透明** | 页面直接展示价格比「联系销售」转化更好；若有年付，可做月/年切换（约 +28% 转化）。 |
| **首屏 CTA 可见** | 移动端主按钮尽量在首屏内可见，无需滚动即可点击。 |
| **单页单目标** | 一页聚焦一个转化目标，避免多个同等权重 CTA 分散注意力。 |
| **Hero 区视频** | 在 Hero 或紧接 Hero 使用**短视频**介绍产品，部分案例中可带来约 86% 的转化提升。 |
| **速度** | LCP < 2.5s；每慢约 1 秒，转化约降 7%。 |

## 二、当前站点对照与功能性优化项

### 2.1 首页

| 功能点 | 当前状态 | 高端站常见做法 | 建议 |
|--------|----------|----------------|--------|
| 首屏价值主张 | 有标题 + 副标题 + 内测标签 | 一句话说清「为谁 + 得到什么」 | 可选：副标题首句更具体，如「电商与自媒体团队的 AI 运营副驾」 |
| 首屏信任信号 | 仅有「内测进行中」 | 首屏 3+ 信任元素（用户数/Logo/评价/保障） | **增加**：主 CTA 下方一行「无需信用卡 · 注册即送积分 · Win & Mac」；若有数据可加「X+ 团队已在用」 |
| 产品演示 | 已有自动播放的对话+生图演示（ProductDemo） | 互动演示优于静态图 | ✅ 已具备；可考虑加「点击切换场景」或「查看功能」跳转 |
| Hero 视频 | 无 | 部分站点在 Hero 用 30–60 秒产品视频 | 可选：在 Hero 与 ProductDemo 之间或使用指南页增加产品演示短视频 |
| 主 CTA | 「立即免费开始」+「查看演示」 | 主 CTA 明确、对比度高 | ✅ 已较好；保持主按钮为「免费开始」 |
| 移动端首屏 CTA | 有，但需确认是否完全在首屏内 | 主按钮首屏可见 | 可选：移动端加**吸底 CTA 条**（滚动过 Hero 后显示「免费下载」），减少滚动才能点到的流失 |
| 信任密度（整页） | Metrics 在下方、Testimonials 有 3 条 | 每屏有 1+ 证明元素 | 可选：在 PromptGallery 前或 Workflow 前加一句「已有 X 类用户在用的案例」或 1–2 个客户 Logo（若有授权） |

### 2.2 下载页

| 功能点 | 当前状态 | 建议 |
|--------|----------|------|
| 主操作 | 按版本/平台选下载 | ✅ 清晰 |
| 安装步骤 | 有 Git + 安装 + 邀请码 + 注册 | 可选：增加「常见安装问题」折叠或链到使用指南 |
| 信任/保障 | 有邀请码与客服二维码 | 可选：加一句「安装遇到问题？联系客服」或链接到使用指南/FAQ |

### 2.3 定价页

| 功能点 | 当前状态 | 建议 |
|--------|----------|------|
| 价格展示 | 对话包 + 图包 + 引擎单价 + 注册即送 | ✅ 透明 |
| 年/月切换 | 无 | 若有年付优惠，可增加「月付 / 年付」切换，提升 LTV 与转化 |
| FAQ | 有 | ✅ 可保留；可选与使用指南、下载页交叉链接 |
| 主 CTA | 「免费开始」到下载页 | ✅ 一致 |

### 2.4 使用指南页

| 功能点 | 当前状态 | 建议 |
|--------|----------|------|
| 教学视频 | 占位 | 上线后首屏或次屏展示，可提升「会用才留」的转化 |
| 图文步骤 | 简要安装与注册 | 可补充「首次打开客户端」「选 Agent / 生图流程」等 3–5 步，并配截图 |
| 与下载页联动 | 有「前往下载」 | 可选：在下载页增加「第一次用？看使用指南」链接 |

### 2.5 功能页 / 关于页

| 功能点 | 建议 |
|--------|------|
| 功能页 | 可选：每个 Agent 卡片增加「查看示例」或「试用该能力」链到下载/演示 |
| 关于页 | 保持简洁；若有客户 Logo 或合作方，可在此或首页增加 Logo 墙 |

### 2.6 全站

| 功能点 | 建议 |
|--------|------|
| 性能 | 图片用 Next/Image、关键资源预加载，保证 LCP < 2.5s |
| 移动端导航 | 展开菜单时禁止背景滚动，避免误触 |
| 统一 CTA | 全站主 CTA 统一为「免费开始」或「免费下载」，副 CTA 为「查看功能」/「联系销售」等，避免多目标分散 |

## 三、建议优先级（可转为 Issue）

| 优先级 | 项目 | 预期影响 |
|--------|------|----------|
| P0 | Hero 主 CTA 下增加一行信任文案（无需信用卡 · 注册即送积分 · Win & Mac） | 降低首屏心理门槛，提升点击意愿 |
| P0 | 使用指南页上线教学视频 + 3–5 步图文（含截图） | 提升「安装后会用」的留存与口碑 |
| P1 | 移动端吸底 CTA（滚动过 Hero 后显示「免费下载」） | 减少移动端因需滚动才点到 CTA 的流失 |
| P1 | 定价页若有年付则增加月/年切换 | 提升客单价与转化 |
| P2 | 首页增加 1 句「已有 X 类用户」或客户 Logo 墙（需有授权） | 提升信任密度 |
| P2 | Hero 或首屏下增加 30–60 秒产品短视频 | 部分场景可显著提升转化 |
| P2 | 下载页/使用指南/定价页之间交叉链接（FAQ、安装问题、首次使用） | 降低跳出、提升完成率 |

## 四、如何用本文件开 Issue

1. 在 GitHub 仓库 **Issues** 点 **New issue**。
2. 选「功能/体验优化」模板，标题如：`[官网] Hero 下增加信任文案`。
3. 描述中可写：对应 **FUNCTIONAL_IMPROVEMENTS.md** 第 X 节、优先级、期望效果。
4. 完成后可在本文件对应项后注明「已做」或「Issue #N」。
